Quick Start

A practical path from credentials to first accepted payload.

Step 01

Request Waterly credentials

Get the submission URL, client token, and assigned device identity from the Waterly team.

Step 02

Choose your runtime

Use the integration directory to select a package, script, or gateway project that fits your environment.

Step 03

Send a minimal payload

Submit tag names, numeric values, and second-based timestamps using the canonical WaterlyConnect schema.

Step 04

Operationalize and monitor

Move from a one-off test to scheduled jobs, gateway events, or device-side timers as appropriate.
