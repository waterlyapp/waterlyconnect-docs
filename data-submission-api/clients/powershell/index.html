Windows-native SDK

PowerShell module for WaterlyConnect operational workflows.

Use the PowerShell module when your submission path already lives in Windows administration tooling, task scheduler jobs, or operator-driven scripts. The module builds device and tag payloads with cmdlet-style helpers and submits them over HTTP.

Best Fit
Operator-driven or scheduled Windows automations that want native PowerShell objects.
Prerequisites

What to have in place

  • PowerShell 5.1+ or PowerShell 7+ in the runtime environment.
  • The imported WaterlyConnect.psm1 module available to the script.
  • Waterly submission URL, client token, and device identity values.
Quick Start

Minimal usage path

  1. 1
    Import the module.
    Load the local WaterlyConnect.psm1 file into your session or script.
  2. 2
    Create the client and device objects.
    Use the helper cmdlets to shape payload-compatible objects.
  3. 3
    Submit a tag array.
    Build timestamped tag entries and call SubmitData.
Example

Submission snippet

Import-Module ./WaterlyConnect.psm1

$clientDevice = New-WaterlyConnectDeviceInfo `
  -Id "booster-station-02" `
  -Type "PowerShell Gateway"

$client = New-WaterlyConnectApiClient `
  -Url "https://example.waterly.com/connect/submit" `
  -ClientToken "your-client-token" `
  -ClientDevice $clientDevice

$tags = @(
  New-WaterlyConnectTagDatum `
    -Name "ClearwellLevel" `
    -Value 14.2 `
    -LastChangeTimestamp ([int][DateTimeOffset]::UtcNow.ToUnixTimeSeconds())
)

$client.SubmitData($tags)
