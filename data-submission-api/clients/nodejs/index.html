SDK Guide

Node.js client for typed WaterlyConnect submissions.

This package is best suited for services, connectors, and ETL jobs that want a small axios-based client plus generated TypeScript schema types sourced from the canonical WaterlyConnect OpenAPI document in this repository.

Prerequisites

What you need before wiring it in

  • Node.js available in the environment that will submit payloads.
  • A Waterly-provided submission URL ending in /connect/submit.
  • A client token and assigned device identity with id and type.
  • Outbound network access from the runtime to the WaterlyConnect endpoint.
Quick Start

Minimal setup path

  1. 1
    Install the package.
    Add @waterclick/waterlyconnect-nodejs-client to your project.
  2. 2
    Create the client config.
    Supply the Waterly URL, token, and device object.
  3. 3
    Submit tag data.
    Send name, numeric value as string, and a seconds-based timestamp.
Example

Submission snippet

import {
  TagDatum,
  WaterlyConnectApiClient,
  WaterlyConnectApiClientConfig
} from "@waterclick/waterlyconnect-nodejs-client";

const config: WaterlyConnectApiClientConfig = {
  url: "https://example.waterly.com/connect/submit",
  clientToken: "your-client-token",
  clientDevice: {
    id: "plant-west-01",
    type: "SCADA Gateway"
  }
};

const client = new WaterlyConnectApiClient(config);

const tags: TagDatum[] = [
  {
    name: "FinishedWaterFlow",
    value: "123.2",
    last_change_timestamp: Math.floor(Date.now() / 1000)
  }
];

await client.submitData(tags);
