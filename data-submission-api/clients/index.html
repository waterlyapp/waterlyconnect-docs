What stays canonical

WaterlyConnect contract and doc flow

  • data-submission-api/waterly-connect.yml is the source of truth.
  • The generated WaterlyConnect HTML reference remains at /data-submission-api/.
  • Imported source packages remain under /data-submission-api/clients/packages/.
  • These wrapper pages provide stable, human-oriented entry points for GitHub Pages.
How to use this page

Pick by deployment environment

  • Use Node.js when you want a typed package in a service or adapter.
  • Use Python when a single-file client fits best.
  • Use PowerShell for Windows-native operational automation.
  • Use Ignition or Ewon Flexy for plant-floor or gateway submissions.
