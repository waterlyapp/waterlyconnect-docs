Gateway Project

Ignition project export for scheduled WaterlyConnect posting.

The Ignition integration ships as an importable project archive and a viewable source mirror. It is designed for gateways that need to read selected tags on a schedule, shape them into WaterlyConnect payloads, and submit them with device credentials managed by the plant or integration team.

Prerequisites

What you need before import

  • Ignition 8.1+ Gateway with Designer access.
  • Tag providers containing the values you plan to post.
  • Waterly credentials for the submission URL, device ID, and device token.
  • Access to gateway scripting and scheduled or timer events.
Quick Start

Implementation flow

  1. 1
    Import the project ZIP.
    Load ignition_to_waterly.zip from the Gateway webpage or Designer.
  2. 2
    Replace the placeholder credentials.
    Edit the Waterly script constants in the imported script library.
  3. 3
    Bind it to a gateway event.
    Use a Scheduled or Timer event to call the helper with selected tag paths.
Gateway Event Example

Minimal scheduled script

def onScheduledEvent():
    tags = [
        "[Sample_Tags]Realistic/Realistic0",
        "[Sample_Tags]Realistic/Realistic2",
    ]

    waterly.sendDataToWaterly(tags)
