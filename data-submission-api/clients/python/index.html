Single-file SDK

Python client for lightweight WaterlyConnect automation.

The Python client is a good fit when you need a single-file module that mirrors the WaterlyConnect payload model without pulling in third-party dependencies. It is especially convenient for utility scripts, cron jobs, and compact adapter services.

Prerequisites

Deployment expectations

  • Python 3.12 or newer in the environment that submits tag data.
  • A WaterlyConnect URL, client token, and device object with id and type.
  • A place to store the module or a PYTHONPATH that points at the imported client directory.
Quick Start

Minimal usage path

  1. 1
    Copy or import the module.
    Use the file directly or add the client directory to PYTHONPATH.
  2. 2
    Create the config object.
    Set URL, token, and device metadata before constructing the client.
  3. 3
    Submit a list of tags.
    Each payload entry needs a unique name, numeric value, and seconds-based timestamp.
Example

Submission snippet

import time

from waterlyconnect_client import (
    ClientDeviceInfo,
    TagDatum,
    WaterlyConnectApiClient,
    WaterlyConnectApiClientConfig,
)

config = WaterlyConnectApiClientConfig(
    url="https://example.waterly.com/connect/submit",
    client_token="your-client-token",
    client_device=ClientDeviceInfo(
        id="lift-station-04",
        type="Python Adapter",
    ),
)

client = WaterlyConnectApiClient(config)

tags = [
    TagDatum(
        name="WetWellLevel",
        value=12.7,
        last_change_timestamp=int(time.time()),
    )
]

client.submit_data(tags)
